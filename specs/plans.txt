================================================================================
                        ECON DESK — Terminal Dashboard
                              Specification v1.0
================================================================================

1. OVERVIEW
--------------------------------------------------------------------------------
A terminal-based real-time economy dashboard that displays live pricing across
major asset classes, macro indicators, and regime-aware Monte Carlo projections.
The system is fully keyboard-driven and runs in the terminal.


2. TRACKED INSTRUMENTS & INDICATORS
--------------------------------------------------------------------------------

  EQUITIES (ETFs)
    - SPY   (S&P 500)
    - IWM   (Russell 2000)
    - QQQ   (Nasdaq 100)
    - DIA   (Dow Jones)

  COMMODITIES
    - Crude Oil (WTI)
    - Gold
    - Silver
    - Natural Gas
    - Copper
    - Wheat

  CRYPTO
    - BTC  (Bitcoin)
    - ETH  (Ethereum)
    - SOL  (Solana)

  FIXED INCOME & RATES
    - US 10-Year Treasury Yield
    - US 2-Year Treasury Yield
    - 2s10s Yield Curve Spread (10Y minus 2Y)

  VOLATILITY & FX
    - VIX (CBOE Volatility Index)
    - DXY (US Dollar Index)

  MACRO INDICATORS
    - Inflation Expectations (e.g. 5Y breakeven)
    - Initial Jobless Claims (weekly)


3. MAIN DASHBOARD VIEW
--------------------------------------------------------------------------------
The default screen shows a table/grid of all tracked instruments with:
  - Current price / level
  - Daily change (absolute & percentage)
  - Color-coded directional signal (green/red)
  - Mini sparkline or trend indicator

Layout is compact, designed for a standard terminal (minimum 120x40).


4. ANALYTICAL PIPELINE
--------------------------------------------------------------------------------
The core analytical engine is a three-stage pipeline:

  STAGE 1 — GLOBAL HIDDEN MARKOV MODEL (HMM)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Inputs:
    - Rolling returns for all tracked assets
    - Macro indicators (yields, spreads, VIX, DXY, inflation expectations,
      jobless claims)

  Outputs:
    - Current macro regime classification (e.g. Risk-On, Risk-Off,
      Transition, Crisis, Reflation — exact states TBD)
    - Regime confidence score (probability of being in current state)
    - Transition probability matrix (probability of moving to each other
      regime within the next 30 days)

  Timeframe: 30-day forward-looking window
  Training window: TBD (rolling historical calibration)

  STAGE 2 — REGIME-CONDITIONED PARAMETERS
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  For the identified regime (weighted by confidence):
    - Compute regime-specific mean return vector (per asset)
    - Compute regime-specific covariance matrix (cross-asset)

  These parameters reflect the statistical behavior of assets under the
  current macro regime, not unconditional historical averages.

  STAGE 3 — MULTIVARIATE MONTE CARLO SIMULATION
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  Inputs:
    - Regime-conditioned mean returns
    - Regime-conditioned covariance matrix

  Process:
    - Draw correlated return paths from the multivariate distribution
    - Simulate 10,000 paths over 30-day forward horizon
    - Respect cross-asset correlation structure so projections are
      internally consistent

  Outputs (per asset):
    - Projection cones (e.g. 10th / 25th / 50th / 75th / 90th percentiles)
    - Expected value path
    - Tail risk metrics

  Key property: Because all assets share the same correlated simulation,
  projection cones are CONSISTENT with each other — if equities project
  down in a risk-off regime, safe havens project accordingly.


5. DRILL-DOWN VIEW (via command)
--------------------------------------------------------------------------------
The user types a command to inspect any instrument (e.g. "view SPY",
"show gold"). Tab auto-completes instrument names. Detail panel shows:

  PRICE INFORMATION
    - Current price
    - Daily high
    - Daily low
    - Volume
    - 24-hour volume change (%)

  PRICE CHART
    - ASCII/terminal-rendered price graph
    - Configurable timeframe (1D, 1W, 1M, 3M, etc.)

  MONTE CARLO PROJECTION CONES
    - Forward projection chart rendered in terminal
    - Percentile bands (10/25/50/75/90)
    - Derived from the global multivariate simulation (Stage 3)
    - Shows how this asset's projection relates to the current regime

  REGIME CONTEXT
    - Current regime label and confidence
    - This asset's regime-conditioned expected return
    - Key correlated assets and their directional alignment


6. INTERACTION MODEL
--------------------------------------------------------------------------------
  - Pure terminal / command-driven
  - Command input with tab auto-completion for instrument names
  - Commands to navigate (e.g. "view <asset>", "refresh", "back")
  - ESC or 'q' → return to main dashboard
  - Manual refresh only (user-triggered via command)
  - No mouse/click interaction — fully keyboard and command based


7. DATA SOURCES
--------------------------------------------------------------------------------

  YAHOO FINANCE (via yfinance Python library)
    - SPY, IWM, QQQ, DIA — price, volume, daily high/low
    - Crude Oil (CL=F), Gold (GC=F), Silver (SI=F), Natural Gas (NG=F),
      Copper (HG=F), Wheat (ZW=F)

    - DXY (DX-Y.NYB)

  COINGECKO API
    - BTC, ETH, SOL
    - Price, volume, 24h volume change

  FRED API (Federal Reserve Economic Data)
    - Fed Funds Rate
    - Inflation expectations (e.g. T5YIE — 5-Year Breakeven)
    - Initial Jobless Claims
    - Yield curve spreads (T10Y2Y — 2s10s spread)

  US TREASURY WEBSITE
    - Daily yield curve rates (2Y, 10Y, and full curve)


8. HMM REGIME DEFINITIONS (5 States)
--------------------------------------------------------------------------------

  1. BULL
     - Growth expansion, risk-on environment
     - Equities trending up, credit spreads tight, moderate volatility
     - Typical: SPY/QQQ rising, VIX low, yields stable or rising with growth

  2. BEAR
     - Contraction, risk-off environment
     - Equities grinding lower, widening spreads, rising defensive positioning
     - Typical: SPY/QQQ declining, bonds bid, DXY mixed

  3. STAGNATION
     - Sideways, low volatility, low growth
     - Range-bound markets, muted returns, low conviction
     - Typical: flat equities, low VIX, stable yields, low volume

  4. STAGFLATION
     - Low growth combined with rising inflation
     - Correlation structure flips: stocks down, bonds also down (no haven),
       commodities up
     - Typical: rising inflation expectations, gold/oil up, equities weak,
       yields rising despite weak growth

  5. CRISIS
     - Acute stress, extreme volatility, liquidity breakdown
     - Correlations spike toward 1, violent drawdowns, panic selling
     - Typical: VIX 35+, sharp equity drops, flight to cash/treasuries,
       credit freezes

  TRAINING
     - Lookback window: 5 years rolling
     - Calibration: batch retrain daily
     - Timeframe: 30-day forward regime classification


9. DEPENDENCIES
--------------------------------------------------------------------------------

  CORE
    - Python 3.x
    - Textual         — terminal UI framework (includes Rich, built-in graphing)
    - NumPy           — numerical computation, covariance matrices,
                        multivariate Monte Carlo (np.random.multivariate_normal)
    - SciPy           — Student's t-distribution (scipy.stats.t) for fat-tailed
                        simulation in Crisis/Bear regimes; also used by hmmlearn
    - hmmlearn        — Gaussian Hidden Markov Model for regime detection

  DATA
    - yfinance        — equities, commodities, VIX, DXY pricing via Yahoo Finance
    - fredapi         — FRED API client for macro indicators
    - CoinGecko API   — crypto pricing (via requests or pycoingecko)

  STORAGE
    - DuckDB          — local columnar database, single file, optimized for
                        analytical queries on time-series data

  STDLIB / TRANSITIVE
    - requests        — HTTP calls (CoinGecko, US Treasury)
    - Rich            — bundled with Textual

  LOGIC FLOW
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  1. main.py launches app.py
  2. app.py boots Textual, initializes DuckDB via store.py, loads dashboard
  3. User types "refresh" → data/*.py fetchers pull latest → store.py
     writes to DuckDB
  4. On refresh: hmm.py retrains on 5 years from DuckDB → regime + transition
     matrix → regime.py extracts conditioned means/covariance →
     montecarlo.py runs 10,000 paths
  5. Dashboard displays results; user types "view SPY" → drilldown.py
     renders detail panel from stored data + MC results

  DATA FETCH STRATEGY (Incremental)
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  1. On first run: DB is empty → fetch full 5 years of history for all
     instruments → write to DuckDB
  2. On every subsequent run: query DuckDB for the latest date per
     instrument → fetch only missing days (last date + 1 to today) →
     append to DuckDB
  3. If latest date is today: skip fetch entirely for that instrument
  4. store.py exposes get_latest_date(instrument) — each fetcher in
     data/ uses this to determine where to pick up from

  SEPARATION OF CONCERNS
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
  - data/   — fetching and storage only; no UI or model logic
  - models/ — math only; no knowledge of data sources or display
  - ui/     — presentation only; calls into data and models
